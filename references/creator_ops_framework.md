# 创作者型 YouTube 运营方法论

This reference captures the channel-operation framework used by this skill.

## 1. 视频是作业，不是作品

The early goal is not perfection. The goal is to build a repeatable system:

- Publish.
- Observe data.
- Identify failure point.
- Revise the next video.

Common failure mode: spending weeks preparing equipment, editing templates, logos, and intros before validating whether viewers care.

## 2. 日更是训练，不是宗教

Daily posting can compress the learning cycle, but only if every post has a hypothesis and a review loop.

Use daily posting when:

- The creator has weak execution rhythm.
- Topic judgment is immature.
- Production cost is low.
- There is enough topic inventory.

Avoid daily posting when:

- Quality collapses.
- Research quality matters.
- Burnout destroys consistency.
- No one reviews data.

## 3. 极简自媒体优先级

Priority order:

1. Topic.
2. Viewpoint.
3. Viewer promise.
4. Opening hook.
5. Title.
6. Thumbnail.
7. Update rhythm.
8. Data review.
9. Gear.
10. Editing polish.

## 4. 不要把学习当执行替代品

Learning should solve the current bottleneck.

Bad pattern:

- Watch more tutorials.
- Buy more tools.
- Redesign brand kit.
- Delay publishing.

Better pattern:

- Publish one test.
- Identify the weakest metric.
- Learn only what fixes that metric.
- Apply immediately.

## 5. 频道定位公式

```text
I help [target viewer] solve [core problem] through [content format] so they get [clear result].
```

## 6. 内容支柱

Every channel needs repeatable content pillars.

Examples for YouTube-operation content:

- Core mindset.
- Practical tutorials.
- Case studies.
- Q&A.
- Risk avoidance.
- Trend interpretation.

## 7. 复盘原则

Views alone are not enough. Diagnose the funnel:

```text
Impressions → CTR → first 30s retention → average view duration → subscription → monetization
```

Each weak point implies a different fix.
